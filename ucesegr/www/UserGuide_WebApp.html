Guide content

The following guide explains how to use the UCL WebApp and to set up new questions for the QuizApp for mobile. Here is the content of the guide:

- System requirements;

- How to start the app;

- How the app works.

The app is developed for technical users who manage QuizApp, in order to set up new questions. It should not be used by final users, since it requires at least a basic technical knowledge in order to be edited.

System requirements

The app simply needs a browser to be run. It works on Google Chrome, Microsoft Internet Explorer, Microsoft Edge, Safari and Mozilla Firefox.

The user must be connected to the UCL VPN in order to load the web page of the app. We recommend the Cisco AnyConnect app to establish the VPN connection; further information about how to connect to the UCL VPN can be found on the following link.

How to start the app

The app is an on-line application available at the WebApp web page . Hence, there is no need to install it on a device.

Before opening the app, the user must be connected to the UCL VPN and must start both the server and the Phonegap server through an SSH software. We recommend Bitvise for the Windows environment, while Fugu SSH is agood alternative for the iOS environment. The connection is established by using the following credentials:

Host: developer.cege.ucl.ac.uk

Port: 30079

Username: studentuser

Password: XXXXXX - contact the administrator for the password

Once the connection starts, the user has to type the following commands on the terminal to make the server run:

- cd /home/studentuser/code (press enter)

- node NodeJSserver.js (press enter - the server is activated and data can be uploaded/downloaded)

- cd WebApp/ucesibi (press enter)

- sudo su (press enter and enter the password)

- phonegap serve (press enter - the Phonegap server is activated and the app can be run)

At this point, the user can finally access to the WebApp web page to set up new questions.

How the app works

The app is developed to be used on a computer, however it can be used also on mobile devices. In this case, it could be less convenient to insert new data.

The WebApp has a main interface which contains a map that can be explored by the user to find new locations to place a point. The coordinates of a specific location can be extracted by clicking on the map, so a pop-up window will show the latitude and the longitude of that point; those can be copied and subsequently pasted in the respective fields in the question form.

The left-side menu has two options:

Add a question: the button will redirect the user to the question form that must be used to create a new question;

Load Points: the button will load the points of interest already uploaded on the server. It could be useful to know where points have already been placed in order to avoid repetitions.

From the top-right menu the user can access both this user guide (User Guide link) and additional documentation that gives more information about the app and its functionalities (More Info link).

The question form contains all the fields required to set up a new question. Once the user has filled all of them, they must click on the Upload data button to upload and to store the new data on the server. To set up another question, the user could go back to the map to check the coordinates and open the question form again.
